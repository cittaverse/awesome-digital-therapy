# 贡献指南 (Contributing Guide)

欢迎为 Awesome Digital Therapy 贡献资源！

---

## 收录标准

在提交之前，请确保您的资源符合以下标准：

### ✅ 必须满足

1. **相关性**：与数字疗法、认知训练、老年护理、AI 健康直接相关
2. **质量**：
   - 论文：同行评审期刊
   - 工具：活跃维护的开源项目（12 个月内有 commit）
   - 报告：知名机构发布
3. **可访问性**：链接有效，内容可公开访问（或明确标注申请流程）

### ❌ 不收录

- 营销内容/软文
- 无临床验证的商业产品
- 已停止维护的项目（24 个月无更新）
- 纯英文资源但有优质中文替代时（我们优先中文）

---

## 提交方式

### 方式 1: GitHub Issue (推荐)

1. 点击 [New Issue](https://github.com/cittaverse/awesome-digital-therapy/issues/new)
2. 选择 "添加资源" 模板
3. 填写资源信息
4. 提交等待审核

### 方式 2: Pull Request

1. Fork 本仓库
2. 在对应分类添加资源（按字母/时间排序）
3. 提交 PR，说明添加理由
4. 等待审核（通常 48 小时内）

## 资源质量评分

我们使用以下评分系统帮助读者判断资源质量：

| 标识 | 含义 | 标准 |
|------|------|------|
| 🟢 推荐 | 高质量 | 顶级期刊/广泛使用/权威机构 |
| 🟡 参考 | 中等质量 | 普通期刊/新兴工具 |
| 🔴 谨慎 | 需验证 | 预印本/早期项目/单一来源 |

**示例**：
```markdown
- **[CittaVerse Pipeline](链接)** 🟢 - 神经符号叙事评估引擎 | 中文 | 临床验证
```

## 审核流程

```
提交 → 自动检查 (链接有效性) → 人工审核 (质量/相关性) → 合并 → 通知
```

**审核时间**：通常 48 小时内

**审核标准**：
1. 链接是否有效
2. 是否符合收录标准
3. 格式是否正确
4. 是否与现有资源重复

---

## 维护职责

作为贡献者，如果您添加的是：

- **开源工具**：请在项目停止维护时提交 PR 标注
- **数据集**：请在访问方式变更时更新
- **报告**：请在有更新版本时补充

---

## 行为准则

请遵守 [Contributor Covenant](https://www.contributor-covenant.org/)：

- 使用包容性语言
- 尊重不同观点
- 建设性批评
- 关注社区共同利益
